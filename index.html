Colors

General

These colors are used across the UI; they make up our family of colors

#51b2ca

This blue is used for point related numbering and highlighting.

ex: points in nameplate, point badges on actions, point highlights in gutter notifications

#eb1b0d

Our official red. This should only be used on call to actions. It does not play nicely with other colors.

ex: points in nameplate, point badges on actions, point highlights in gutter notifications

#b75a5a

This maroon-ish red is used where we'd normally use our official red if that color played nicely with other colors. This color doesn't have as strong an association as blue does with points, so there is more freedom using this color with elements. Links will eventually be in this color.

#d3cfc6

The base color for our background (without texture).

#ebe6dc

The base color for our background (without texture). This color is used for contrast against the background.

#444444

No element should be darker than #444, without good reason. Prizes, is a good example of an area that has darker colors.

#f9f9f9

Our "white" is actually an extremely light gray.

Text

These colors will generally get you where you need to go in most text layout situations. These will work well on a white (or similar) background or on either of our beige textured backgrounds.

#444444

This is the darkest black text should be set in, unless legibility is poor.

#777777

This blue is used for point related numbering and highlighting.

ex: points in nameplate, point badges on actions, point highlights in gutter notifications

In some situations, text may be set on a background that creates too much/little contrast with the standard text colors (usually #444 will appear too dark on these backgrounds or #777 will not be legible). In these cases, choosing a color that is close to the intended color in value but close to the background color in hue should preserve the contrast and hierarchy while also fitting into the design and element they are set on.

Typography

General

Crowdtap employs three typefaces on the member side to create a variety of hierarchy options, groupings, and pairings that are clear and balanced. These typographic guidelines will help when setting areas of text on a page.

Rokkitt

General headline/heading text—used for display.

ex: sectional heading on forms, headline copy on modal windows, headline/sub headline on nameplate

Open Sans

Body copy. The majority of our copy should be set in Open Sans--should be default text.

ex: Quickhit answers, button text, nav, brand page active/complete tabs for modules, disclaimer text, regular copy on brand page and modals

Oswald

Sectional heading when Rokkitt is not available/appropriate or as a major heading for a module/contained page element. Oswald should be used sparingly and almost exclusively as a heading label--the main exception being numbers that represent components of our game mechanics.

ex: Quickhit answers, button text, nav, brand page active/complete tabs for modules, disclaimer text, regular copy on brand page and modals

Sizes

The only hard rule for font sizing is copy that is meant to be read at length should not be smaller than 16px. If there is a reason to reduce the font size then 14px is the absolute baseline for body copy in a given area. Small text can be seen in places but these are areas of text that are short (gutter notification copy) or not meant to be read (disclaimer copy).

Pairings

Pairings the fonts in our family is an important detail to pay attention to. Good typography will keep the text on pages from feeling the same and make digesting and reading the page easier.
