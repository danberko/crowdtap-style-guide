Colors

General

These colors are used across the UI; they make up our family of colors

#51b2ca

This blue is used for point related numbering and highlighting.

ex: points in nameplate, point badges on actions, point highlights in gutter notifications

#eb1b0d

Our official red. This should only be used on call to actions. It does not play nicely with other colors.

#b75a5a

This maroon-ish red is used where we'd normally use our official red if that color played nicely with other colors. This color doesn't have as strong an association as blue does with points, so there is more freedom using this color with elements. Inline text links should be this color.

#d3cfc6

The base color for our background (without texture).

#ebe6dc

The base color for the background of the nameplate. This color is used as a background and can contrast well against our darker, textured background.

#444444

No element should be darker than #444, without good reason. Prizes, is a good example of an area that has darker colors.

#f9f9f9

Our "white" is actually an extremely light gray. Large content areas, like the modules on the brand page, use this as a background.

Text

These colors will generally get you where you need to go in most text layout situations. These will work well on a white (or similar) background or on either of our beige textured backgrounds.

#444444

This is the darkest black text should be set in, unless legibility is poor.

#777777

This blue is used for point related numbering and highlighting.

ex: points in nameplate, point badges on actions, point highlights in gutter notifications

In some situations, text may be set on a background that creates too much/little contrast with the standard text colors (usually #444 will appear too dark on these backgrounds or #777 will not be legible). In these cases, choosing a color that is close to the intended color in value but close to the background color in hue should preserve the contrast and hierarchy.

Typography

General

Crowdtap employs three typefaces on the member side to create a variety of hierarchy options, groupings, and pairings that are clear and balanced. These typographic guidelines will help when setting areas of text on a page.

Rokkitt

General headline/heading text—used for display.

ex: sectional heading on forms, headline copy on modal windows, headline/sub headline on nameplate

Source Sans Pro

The majority of our copy is set in Source Sans Pro.

ex: Quickhit answers, button text, nav, brand page active/complete tabs for modules, disclaimer text, regular copy on brand page and modals

Oswald

Sectional heading when Rokkitt is not available/appropriate or as a major heading for a module/contained page element. Oswald should be used sparingly and almost exclusively as a heading label--the main exception being numbers that represent components of our game mechanics.

ex: Quickhit answers, button text, nav, brand page active/complete tabs for modules, disclaimer text, regular copy on brand page and modals

Sizes

The only hard rule for font sizing is copy that is meant to be read at length should not be smaller than 16px. If there is a reason to reduce the font size then 14px is the absolute baseline for body copy in a given area. Small text can be seen in places but these are areas of text that are short (gutter notification copy) or not meant to be read (disclaimer copy).

Pairings

Buttons

General

Sticking to a consistent family of buttons will allow for a button to hint at its functionality simply based on the type of button it is.

Primary Call to Action

The standard Crowdtap call to action is a big red button; it's really hard to miss. It's normally as wide as its content unless in its in a modal window, in which case its the same width as the content in the modal.

Details and Flourishes

General

These are some of the smaller details that help tie elements of the visual system together.

Text Shadows

Text shadow is used to create a "letterpress" effect which helps increase legibility in situations where text is on a background that is noisy—photos, textures, etc. Text shadows also prevent our text from feeling like it's floating on its background.

Box Shadows

Box shadows are used on modules that sit on the page. Shadows are generally black with low opacity and a small, 3 pixel spread. The shadow is meant to be a subtle detail to further separate containers from the background.

Border Radii

We use a border radius of 3px on anything that has a rounded corner (which ends up being just about everything).
